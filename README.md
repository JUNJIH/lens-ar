# LENS — 看板が、見る人によって変わる（第二段階：擬似AR）

スマホのカメラをマーカーにかざすと、その上に「自分向けの飲食店情報」が浮かぶ。
プロフィールを切り替えると、同じマーカーでも中身が変わる。

## これは何の段階か
- 第一段階（済）：カメラなしのモック。切り替えで見え方が変わる面白さを検証。
- 第二段階（今ここ）：実機のカメラで、マーカーにかざすと情報が浮かぶ。
- 第三段階（未）：本物の画像認識で、いろんな看板に対応。

認識は今は「hiro マーカー1種類だけ」に絞ってごまかしている。
まずは実機で「動いた！」を体験するのが目的。

## 動かす手順

### 1. GitHub に置く
このフォルダごと（index.html / app.js / README.md）を新しいリポジトリに push する。

### 2. Vercel でデプロイ
- Vercel で「New Project」→ さっきの GitHub リポジトリを選ぶ
- フレームワークは「Other」でOK（ビルド不要の素のHTML）
- Deploy を押すと、https://～.vercel.app のURLがもらえる

※ カメラは HTTPS でないと動かない。Vercel は自動でHTTPSなので問題なし。

### 3. マーカーを用意
「hiro マーカー」で画像検索して、出てきた黒枠の画像を
スマホの画面に出すか、紙に印刷する。（"AR.js hiro marker" で検索）

### 4. 実機で開く
スマホのブラウザ（Safari / Chrome）で Vercel のURLを開く
→ カメラの許可を求められるので許可
→ hiro マーカーにカメラを向ける
→ マーカーの上に情報パネルが浮かべば成功

### 5. 切り替えてみる
画面下のチップでプロフィールを切り替えると、
同じマーカーでも浮かぶ情報が変わる。これが体験の核。

## 次に中身を読むときの地図
- `index.html` … カメラ起動とマーカー設定（AR.jsの部分）
- `app.js` … プロフィールデータと、パネルを組み立てる部分（← 面白いのはここ）

まず動かして、次の朝の時間に app.js の renderPanel を一緒に読み解く。
